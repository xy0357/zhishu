## 项目总览

### 项目名称
知枢：企业知识资产管理与智能检索平台

### 项目定位
面向企业内部知识资产管理、检索增强问答、文档版本追溯、FAQ 运营与 Agent 审计留痕的一体化平台。

### 当前技术栈
- 前端：React + TypeScript + Vite
- 后端：Rust + Axum + Tokio + SQLx
- 数据层：MySQL 8.0 + Redis 7 + Qdrant + MinIO
- 智能层：OpenAI-Compatible API

### 当前交付基线
- 已有闭环设计文档：[知枢_企业知识资产管理与智能检索平台_闭环落地版_修订稿.docx](</E:/知枢/知枢_企业知识资产管理与智能检索平台_闭环落地版_修订稿.docx>)
- 已有前后端工程骨架与可运行 MVP
- 已有 MySQL DDL、Docker Compose、接口示例、交付说明
- 已有登录、角色分权、用户管理、内容管理、问答演示链路

## 总目标

将当前项目从“可演示 MVP+”推进为“可完美交付版本”，满足以下结果：

1. 功能完整：
   - 文档上传、文档版本、分类标签、FAQ、阅读收藏、问答、引用证据、Agent 审计、用户与角色后台全部闭环。
2. 数据完整：
   - MySQL、Qdrant、MinIO、Redis 全部接入并实际运行。
3. 安全完整：
   - 登录鉴权、密码策略、权限矩阵、令牌机制、密码重置、审计留痕达到正式系统可交付标准。
4. 工程完整：
   - 配置、迁移、脚本、部署、日志、监控、测试、回滚、交付说明齐全。
5. 验收完整：
   - 具备可执行的验收清单、演示路径、测试证据、启动文档与最终交付包。

## 开发任务总原则

- 所有任务必须按顺序推进，前置依赖未完成时，不进入后续阶段。
- `P0` 表示必须完成，否则无法交付。
- `P1` 表示强烈建议完成，否则无法达到“完美交付”。
- `P2` 表示增强项，用于提升可维护性、体验和上线质量。
- 每个任务完成后，同步更新 `PROJECT_BRIEF.md`、`PAGE_MAP.md`、`DATA_MAP.md`、`CHANGELOG.md`。
